This is DHTML Menu Builder's main dialog.
It is the primary application window and is where you create the structure of your menus. You also use it to set the most basic features like the caption of your items, the pages they link to, etc.

Click any part of the dialog to get information about its purpose.

Menu Bar
As with most Windows applications, the menu bar contains the commands that are available in DHTML Menu Builder.

File: Provides an interface for the primary operations applied to a file. (Open, Save, Import, etc.)
Edit: Offers general-purpose editing commands. (Copy, Paste, Find, Delete, Rename, Undo/Redo, etc.)
Menu: Allows access to all the dialogs used to create the menus and change the styles of your items, such as color, font, etcetera.
Tools: Contains utilities that enable you to install your menus, compile your project, preview it and publish it. It also enables access to all the editors integrated into DHTML Menu Builder. (HotSpots Editor, Toolbars Editor, Addin Editor)
Help: Contains commands that provide access to Help information as well as important and useful resources regarding DHTML Menu Builder.

Main Toolbar
This toolbar permits access to the most frequently used functions of DHTML Menu Builder.

New: Creates a new project based on the program's default values.
Open: Opens a standard dialog to locate and open a previously saved project. Clicking the small arrow to the right of the Open button provides immediate access to the last four opened projects.
Save: Saves the current project.
Add Group: Adds a new group to the current project.
Add SubGroup: Adds a new subgroup (or submenu) to the current project. Note that this option is only available when viewing the project in Map View.
Add Command: Adds a new command to the current project.
Add Separator: Adds a new separator to the current project.
Copy/Paste: Copies some or all the styles of an item and pastes them to another item or set of items.
Find: Opens the Find/Replace dialog.
Undo/Redo: Allows you to undo or redo your actions.
Delete: Deletes the selected item. If the selected item is a group, all of its commands will be deleted as well.

Style Toolbar.
This toolbar allows rapid access to all the dialogs used to change the different appearance style options of the current project's items.

Color: Changes all the color-related settings of your items.
Groups:  Changes background color and border color. On the Toolbar tab, you can also change the text color and background color of the hotspots.
Commands: Changes the text color and background color.
Separators: Changes the color of the separator line.
Font: Changes the font and its style used by the items.
Groups: Changes the default font used by the child commands. When using a toolbar, you can also change the font of the hotspots.
Commands: Changes the font and font style.
Cursor: Sets the appearance of the mouse pointer when it moves over the selected item.
Image: Changes the various images associated with an item.
Groups: Changes background color, border color, border width, and background image. When using a toolbar you can also change the image(s) that appear(s) to the left and/or right of the hotspot.
Commands: Change the left, right and background image as well as the the image(s) that appear(s) to the left and/or right of the command.
Leading: Adjusts the amount of space between commands inside a group.
Margins: Adjusts the width of the margin between the inner border of the group and its contents.
Selection Effects: Sets the effects and colors of the bounding rectangle in the menu items.
Special Effects: Sets special effects to spice up your menus (highlighting when the commands are selected, drop shadows, transparency effects, context menus).
Up/Down: Use these buttons to move items up or down. Moving commands up/down will change their position in their respective groups. Moving groups up/down will change their position in their respective toolbar. Moving toolbars up/down will change their position the zIndex order causing that those toolbars at the topmost position will be rendered first and those at the end of the list will be rendered last. The last a toolbar is rendered the greater its zIndex and consequently will appear above any other other toolbar. In case of having overlapping toolbars, those rendered last will appear to be over those rendered first.

Tools Toolbar.
Contains the most important features related to the production of the menus.

Compile: Compiles the current project -- this is what generates the javascript files that contain all the code used to generate and manage the menus.

Publish: Publishes the current project via FTP.
Preview: Displays a preview of the current project.
HotSpots Editor: Opens the HotSpots Editor.
Toolbars Editor: Opens the Toolbars Editor.
Project Properties: Displays the Project Properties dialog.

Menus View.
This pane is the workspace where you design and organize your project items.

Each item is displayed so that you can visualize your groups, commands and separators in an easy way.

Groups are represented by the icon, commands by the icon and separators by the icon.
DHTML Menu Builder will also add several indicators to the items' icons to let you know which events are configured and active.

A number zero "0" under the icon means that the item will response to the mouse when its moved over it.
A number one "1" under the icon indicates that the item will response when the item is clicked.
And a number two "2" indicates that the item will perform some action when its double-clicked.

An arrow displayed below some of these numbers means that the action is to display another group.

See also: View Modes, Enhanced Map View

Caption
Indicates the actual text that will be displayed when you view your menus using a browser.

The Caption is very flexible and supports HTML code. By adding HTML code, you can apply extended formatting to your items' Caption.
For example, you could use the <br> tag to add a new line to the caption, or use the <sup></sup> tags to make a character or word appear as superscript.

Enabled
When not checked, the menu item will not respond to mouse events. (Default is checked.)

This is very useful in creating items that display some information but don't perform any action. You can also use this feature to mimic a disabled item.

Events
The items created by DHTML Menu Builder can support three types of events:

Over: Triggered when the mouse is moved over the item.

Click: Triggered when the item is clicked.
Double-Click: Triggered when the item is double-clicked.

Action
Determines the action that will occur when the selected item is triggered.

DHTML Menu Builder supports four different types of events:

(none): No action is performed. The item will not respond to the selected event.

Open URL / Execute Script: The item will open the linked document or will execute a script. To execute some script, you must precede the code with either JavaScript: or vbscript: (don't forget the colons).
Display Submenu: The item will display another group. This is how you create submenus.
Open URL in new Window: The item will display the linked document in a new browser window. You can specify the new window settings by clicking New Window Settings icon.

URL / Script or Target Menu
This setting specifies what the item must do when the selected event is triggered.
Next to the URL/Script textbox you will find three buttons to help you set your links:

  • Browse: Click to display the Open Dialog. This will let you select the document to which you want to link.
  • Bookmark: Click to access the Bookmark dialog. This will let you hyperlink to named sections on the linked document.
  • Target Frame: Click button to access the Target Frame dialog. This will let you determine the frame in which to load the linked document.

When the event type is set to Open URL / Execute Script you can perform several types of actions:

  • Open a document. To open (or link to) a document simply use the browse button () to select the document that you want to open when the selected menu item is clicked.
     
  • Execute a script or call a javascript function. To write a small javascript function simply prefix your function with the javascript: protocol. For example, to display a dialog box and show a message you would use this code:
    javascript:alert("Hello World");
    Remember to set the Target Frame to "_self" when using this protocol.
     
  • Send an email. To send an email simply prefix the receiver's email address with the mailto: protocol selector. For example, to send an email to jon@doe.com you would use this code:
    mailto:jon@doe.com
    Remember to set the Target Frame to "_self" when using this protocol.

When the event type is set to Display Submenu a list of potential target menus will be displayed. Selecting the target menu will cause it to appear in the box.

When the event type is set to Open URL in new Window the specified URL will be displayed in a new browser window and the the New Window Settings button will be enabled to let you set the new window properties.

Status Text
Text that will be displayed on the browser's status bar.
You can optionally select to display this same text as a tooltip by enabling the "Display Status Text as a tooltip" option from the Global Settings dialog under the Extended Features section.

The status text also accepts two variables: %c and %u
Whenever you write %c, it will be changed by the item's caption and %u will be changed by the item's URL.

Alignment
Specifies the alignment of the displayed menu relative to the hotspot.

Live Preview
Allows you to see how the selected item will appear in your browser.

The Live Preview is updated at the same time you make changes to your items. That way, you always know how your menus will appear without having to go through the process of compiling your project or even previewing.

When a group is selected, the Live Preview shows how the background of the menu will appear. If the toolbar option is selected, the preview will show how the actual toolbar hotspot will appear.

When a command is selected, the Live Preview shows how the selected command will appear against its parent group.

Status Bar
The left side of the status bar provides updated information about the selected item. The right side shows the currently selected (default) configuration. Double-clicking the right section allows you to change the default configuration. The status bar is also used to display progress when the menus are being compiled.

Commands Layout
Defines the way the commands are positioned. This setting applies to groups only:

Vertical: The commands will be placed one below the other, forming a vertical menu.
Horizontal: The commands will be placed one next to the other, forming an horizontal menu.

View Modes
Controls the way DHTML Menu Builder presents your project's menu structure.

Normal: presents all the groups one beneath the other as independent menus. This is the way DHTML Menu Builder normally displays the menus.

Map: presents the menus and submenus in a tree-like structure. Click here for more information.

Splitter
Drag the splitter to resize the Tree and Properties panes.

Compile
When disabled, the selected menu item will not be compiled.

This can be useful if you want to exclude some menu items from your project but without deleting them.
This option will work over toolbars, groups, commands and separators.
