This is the FTP Information section of the Project Properties dialog.
Use this section to enter information about your FTP server so that you can easily publish the files generated by DHTML Menu Builder to your server.
To access this dialog click File->Project Properties, click on the Configurations tab and then click on the FTP Information button.

Click any part of the dialog to get information about its purpose.

Hostname of the FTP server
This is the address of the FTP server.

Absolute path to the server's root folder
This is the absolute path from the FTP server's root folder to the folder where the HTML documents of your web site are stored.

Account Information
Fill in the Username and Password fields with the FTP server's account information.
This section is optional. If some information in this section is omitted, you will be prompted to supply it when you Publish your project.

Use Proxy Server
Enable this option so that communication to the FTP server is performed through a proxy server.

Folder to store compiled files.
This is the folder, from the root web, where you will publish the compiled JavaScript files generated by DHTML Menu Builder.

Folder to store the images.
This is the folder, from the root web, where you will publish the images and sounds used by your menus.

Section buttons
These buttons access the different sections of the Configurations dialog:

  • Paths
  • Target Document
  • Frames Support
  • FTP Information

The HotSpots Editor and Frames Support sections are ignored on configurations of type remote as the information is automatically imported from the associated local configuration.

The Options button provides access to options that allows to:

  • Create new configurations (Options->Add)
  • Edit the selected Configuration (Options->Edit)
  • Remove the selected Configuration (Options->Remove)
  • Set the selected Configurations as the default one (Options->Set As Default)

Another option available on the Options button menu is "Optimize Paths".
When this option is selected, the domain information (Root Web) is removed from all the generated paths. If this option is unchecked on remote configurations, the paths will maintain the Root Web.

For example, if your Root Web points to http://mydomain.com/myweb/ and the Folder to Store Compiled Files points to /menus/ the generated paths will be:

  • Optimization turned on: /myweb/menus/
  • Optimization turned off: http://mydomain.com/myweb/menus/
