Extended Features
By checking the box next to each item, you can add features that affect the way your project is compiled and the way the menus behave:

  • Generate Code for DOM compliant browsers: generate code compatible with Internet Explorer 4 (and above), Navigator 6 and Opera 5.12
  • Generate Code for Netscape Navigator 4: generate special code that works exclusively with the non-standardized HTML, DHTML, CSS and JavaScript implementations of Netscape Navigator 4.
  • Generate hRef file: re-generate this file every time the project is recompiled.
  • Hide Form Elements: hide any dropdown combos or other form elements on the page when a menu is about to be displayed. Because of the way the browsers work, these dropdowns (or pull-downs as they are also called) will always appear over the menus. Hiding the dropdowns when the menus are displayed is the only solution.
Compatibility notes
This option has no effect under Netscape Navigator 4
  • Support for Dreamweaver/Fireworks Rollovers: keeps the menus synchronized with the image-based hotspot rollover effects created with Dreamweaver or Fireworks.
Important Information
If some hotspots display menus and other hotspots do not, you must manually modify the onmouseover event of the hotspots that do not display any menus to prevent the rollover effects from causing problems.

The default onmouseover event may look like this:

onmouseover="MM_swapImage(..........)";

The above event definition must be modified to add a call to the HideAll() function, as displayed on the line of code below:

onmouseover="HideAll();MM_swapImage(..........)";

Also note that when enabling this option, the Root Menus Delay parameter will be ignored.
  • Fix for the Netscape Navigator 4 'Clip Bug': fixes the problem where the menus tend to disappear or appear out of sync with the hotspots under Netscape Navigator 4.
  • Support for ImageReady Rollovers: keeps the menus synchronized with the image-based hotspot rollover effects created with ImageReady.
Important Information
If some hotspots display menus and other hotspots do not, you must manually modify the onmouseover event of the hotspots that do not display any menus to prevent the rollover effects from causing problems.

The default onmouseover event may look like this:

onmouseover="changeImage(..........)";

The above event definition must be modified to add a call to the HideAll() function, as displayed on the line of code below:

onmouseover="HideAll();changeImage(..........)";

Also note that when enabling this option, the Root Menus Delay parameter will be ignored.
  • Lotus Domino Support: enable this feature if the menus are going to be installed in a web server running Lotus Domino.
     
    Important Information

    This option only changes the generated URLs referring the images included within the menus. When this option is checked, any generated link to an image like "<img src=image.gif>" is replaced by "<img src=image.gif/$file/image.gif>", which is domino's extended URL syntax.

    To make DHTML Menu Builder's menus usable within a domino web database, you normally use a web view, e.g. named "pages", which contains all your documents. To provide common pagenames like "index.htm", each document must have such a pagename (e.g. simply use a text field). The page name must be listed in the "pages" view's first sorted column. So you can access your home page document by e.g. "http://my.server.com/path/database.nsf/pages/index.htm".

    To have DHTML Menu Builder's menus now show the used images, create a separate notes document for each image, and attach each single image as a file attachment. To name the document, again set a pagename, filled with the name of the image attached. Those image documents have to be listed in the "pages" view, too! So you can refer to an image e.g. named "image.gif" by "http://my.server.com/path/database.nsf/pages/image.gif/$file/image.gif".
     
  • Item Highlight Functionality: allows the implementation of item highlighting. This will let you highlight menu items to reflect the section where the user is inside the web site.
  • Keyboard Navigation Support: when this option is enabled your will be able to navigate the menus using the keyboard and the mouse:
    • Use the CTRL key on the keyboard to toggle the keyboard support
    • Use the ARROW keys to navigate the menus
    • Use the ENTER key to select a menu item
       
    Important Information
    The keyboard support is limited to the following browsers:
    • Internet Explorer 5, 5.5. and 6 (Windows)
    • Internet Explorer 5 (Mac)
    • Opera 7 (not 100% stable) (Windows)
    • All Mozilla-based browsers (Windows)

    The support is not available under these popular browsers:

    • Netscape Navigator 4
    • Safari
    • Konqueror
  • Display Status Text as a tooltip: enabling this option will display the Status Text defined on a menu item as a tooltip.
  • Enable Global In-Group Scrolling: when this option is enabled, DHTML Menu Builder will displaying the scrolling buttons on large menus but only when necessary, that is, when the group is so large that it cannot feet inside the visible area of the browser.
    This is done automatically and in real-time so if the user resizes the browse window, DHTML Menu Builder will automatically add or remove the scrolling buttons on those menus that require them.
  • Enable Search Engine Optimizations: the menus generated by DHTML Menu Builder are not visible to search engines so search engines cannot properly index a web site where the menus are the only mean of navigation. Enabling this option will cause DHTML Menu Builder to install a plain HTML-based navigation version of the menus which is 100% search engine friendly yet invisible to your web site visitors.
    Note that for this option to work you must re-install the menus. This is also true if you decide to disable the option; changes to this option are not applied by re-compiling the project but by re-install the menus.
